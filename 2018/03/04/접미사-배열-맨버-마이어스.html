접미사 배열

  • 접미사 배열의 길이는 항상 H이므로 문자열 탐색 시에 접미사 배열 내부의 아이템을 찾는 시간은 logH, 문자열 검색에 N
  • 그래서 시간 복잡도는 O(NlogH)

접미사 배열 생성 - 단순한 알고리즘

    struct comparator {
        const string& source;
        comparator(const string& s):source(s){}
        bool operator()(int i, int j) {
            return strcmp(source.c_str()+1, source.c_str() + j) < 0 ;
        }
    }

    vector<int> getSuffixArray(const string & source){
        vector<int> output;
        for(int i = 0 ; i < source.size(); i++) {
            output.push_back(i);
        }
        sort(output.begin(), output.end(), comparator(source));
        return output;
    }
  • 시간복잡도: 문자열 비교에 N, 정렬에 NlogN, 시간복잡도: O(N2logN)

맨버-마이어스 알고리즘

  • 접미사 목록을 여러번에 걸쳐서 정렬
  • 처음에는 한글자 기준, 다음에는 두 글자 기준, .. 등등으로 여러번에 걸쳐서 정렬함
